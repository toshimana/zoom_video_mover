# 人の判断が必要な事項ガイドライン - Zoom Video Mover

**適用範囲**: Claude Code Assistant 支援開発
**目的**: 効果的な人間-AI協働の実現

## 概要

Claude Code Assistantが効果的に動作するために、人からの明確な判断が必要となる事項の基準を定める。

## 人間判断が必要な主要カテゴリ

### 1. アーキテクチャ・設計判断
- **緊急**: 新しい技術選択、破壊的変更、セキュリティレベル設定
- **重要**: パフォーマンス vs 可読性のトレードオフ、API設計
- **通常**: 実装パターン選択、依存関係選定、設定デフォルト値

### 2. 機能仕様・要件判断
- **緊急**: 仕様の矛盾・不明確点の解決、スコープ変更
- **重要**: 優先度設定、エラー処理方針、非機能要件
- **通常**: デフォルト値、バリデーション厳密性、ログ出力詳細度

### 3. 実装戦略・優先度判断
- **緊急**: 技術負債対応、重大な性能問題
- **重要**: 機能実装 vs バグ修正のバランス、テスト戦略
- **通常**: コード組織、命名規則

### 4. セキュリティ・プライバシー判断
- **緊急**: データ保護方法、外部通信セキュリティ
- **重要**: ログ出力範囲、一時ファイル管理、権限管理
- **通常**: セッションタイムアウト、暗号化アルゴリズム選択

## 判断を求める際のフォーマット

```
## 判断が必要な事項：[事項名]

### 背景・状況
[現在の状況、発生した問題]

### 選択肢
A) [選択肢1] - [メリット/デメリット]
B) [選択肢2] - [メリット/デメリット]

### 影響範囲
- 実装への影響：[変更箇所]
- ユーザーへの影響：[体験の変化]

### 推奨案
[Claudeの推奨とその理由]
```

## 自動判断可能な事項

Claude Code Assistantが自動判断できる範囲：
- 一般的なコーディング規約（Rust標準パターン）
- 既存パターンに沿った実装
- 明確な仕様に基づく実装
- バグ修正、コードリファクタリング
- テスト・ドキュメント追加

### 前提条件
- CLAUDE.mdの規約に従った実装
- 既存コードパターンとの整合性
- 業界標準のベストプラクティス適用

## 効果的な指示のコツ

### 良い指示例
```
OAuth認証失敗時に、ユーザーに再認証を促すダイアログを表示し、
3回失敗したらアプリケーションを終了する処理を実装して
```

### 避けるべき指示例
```
エラー処理を改善して
```

### 効果的な指示の要素
1. **具体的な動作**: 何をどのように実装するか
2. **条件・制約**: 実装時の制限事項
3. **期待結果**: 実装後の動作・状態
4. **品質基準**: 満たすべき要件
